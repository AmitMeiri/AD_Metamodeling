# SuStaIn 2021 Metamodeling & Surrogate Pipeline

This repository contains an end-to-end simulation of Alzheimer's Disease progression using the **pySuStaIn** (Subtype and Stage Inference) framework, originally introduced in the paper:
> **"Four distinct trajectories of tau deposition identified in Alzheimer’s disease"** (Vogel et al., Nature Communications).
> [Nature Article Link](https://www.nature.com/articles/s41591-021-01309-6) | [Original pySuStaIn GitHub](https://github.com/ucl-pond/pySuStaIn)

Alongside the original model code, this directory includes a **Surrogate Model** implementation designed to replace computationally expensive Monte Carlo (MCMC) simulations with an instant, probability-based Neural Posterior Estimator.

---

## About the Model

The original SuStaIn model uses longitudinal or cross-sectional patient data to map the temporal progression of disease biomarkers into distinct trajectories (subtypes). It models how different regions of the brain accumulate pathology (e.g., Tau) over time. 

**Inputs:**
- Cross-sectional regional biomarker measurements (e.g., continuous Tau PET Z-scores across various brain regions).

**Outputs:**
- **Subtype Probabilities** (`prob_subtype`): The posterior probability distribution of the patient belonging to each disease subtype/trajectory (e.g., "Classic Limbic").
- **Stage Probabilities** (`prob_stage`): The posterior probability distribution across all possible progression stages ($0$ to $N$).
  - *Surrogate Note:* While the original base model outputs a full discrete probability distribution array across all $N$ stages, our Surrogate Model in this folder collapses this into a single continuous scalar called `expected_stage`. It does this by calculating the mathematical **Expected Value** (the weighted average: $\sum_{i=0}^{N} i \times P(\text{stage}_i)$). This continuous format is essential because our Normalizing Flow networks require continuous variables to operate efficiently.


**Training:**
- The model is trained using a Bayesian Markov Chain Monte Carlo (MCMC) algorithm that automatically discovers the optimal number of subtypes and the sequence of biomarker abnormalities for each subtype from the raw data.

> [!NOTE]
> **The Three Discovered Trajectories (Subtypes):**
> In our included simulation, the optimal model discovered three distinct pathways for Tau spread:
> 1. **Subtype 1 (Neocortical-Leaning/Atypical, 36.3%):** Tau begins outside the classic limbic center (e.g., Precuneus, temporal neocortex). Clinically associated with atypical phenotypes where executive or language deficits appear early.
> 2. **Subtype 2 (Classic Limbic, 49.5%):** The classic Braak pathway originating strictly in the medial temporal lobe (Entorhinal cortex). Associated with early severe memory loss and a slower, plateauing overall disease velocity.
> 3. **Subtype 3 (Neocortical-Leaning/Atypical, 14.2%):** Another atypical variant originating in the Middle Temporal gyrus, spreading aggressively into the neocortex similar to Subtype 1.
> *Note: These specific subtypes ($P_0, P_1, P_2$) are critically important later when coupling this model to the ODE Temporal model, as the patient's predicted subtype will be used to mathematically influence their longitudinal velocity.*

> [!WARNING]
> **Simulated Data Disclaimer:** The data used in this specific repository is **synthetically simulated**. Therefore, the temporal trajectories and subtypes generated by the scripts here *do not* reflect the exact real-world biological data found in the original paper, though they mathematically resemble them for testing and metamodeling purposes.

## Directory Overview

This folder contains three distinct sets of files:
1. **The Base SuStaIn Scripts:** Original files to simulate data and run the SuStaIn MCMC algorithm.
2. **The Base SuStaIn Artifacts:** Pre-calculated posterior parameter matrices from rigorous MCMC training.
3. **The Surrogate Module (`Surrogate_SuStaIn/`):** A sub-directory dedicated to **Surrogate Modeling**. Because fitting SuStaIn or classifying patients using standard MCMC can be computationally heavy, we have built a Neural Posterior Estimation (NPE) surrogate. This neural network acts as a drop-in replacement that instantly outputs the exact mathematical probability distribution of a patient's subtype and stage.

## File & Folder Map

### Base SuStaIn Files
* **[`walkthrough_base_model.md`](walkthrough_base_model.md)**: **Start here for the math and theory!** A detailed technical guide explaining the biological background, the structure of the trained `.pickle` files, the parameters, priors, and how to compute posteriors.
* `run_simulation.py`: The main simulation script. It generates synthetic data, runs the full MCMC inference, and plots the results.
* `sim_zscore_output/tau_sim_10k_simulated_data.pickle`: The original synthetic patient dataset generated by the simulation.
* `sim_zscore_output/pickle_files/tau_sim_10k_subtype2.pickle`: **The actual trained SuStaIn model.** This specific file contains the optimal 3-subtype parameter matrices (posteriors, fractions, sequences) used later to train our Surrogate Model.
* `sim_zscore_output/` also contains other cached outputs (e.g. models for 1 or 2 subtypes) and generated analytical plots (like `PVD_estimated.png`).
* `pySuStaIn/`: The core model library (see Requirements below).

### Surrogate Files (`Surrogate_SuStaIn/` directory)
* **[`walkthrough_surrogate_model.md`](walkthrough_surrogate_model.md)**: **Start here for the Surrogate!** The walkthrough guide explaining the surrogate architecture (mapping regional Z-scores to Subtype & Stage distributions), and how to train/validate the NPE network.
* `build_training_data.py`: Extracts and prepares data from the original SuStaIn output into a JSON format.
* `preprocess_surrogate.py`: The main trainer script that fits the flow-based NPE surrogate.
* **`tmp/surrogate_artifacts/`**: **Where the trained Surrogate Model is saved!** The external `bayesian_metamodeling` library automatically saves the fully trained surrogate network (`backend_payload.json`) into this directory. When running fast inference, you load the surrogate from here.
* `validate_test_subject.py` & `plot_surrogate_figures.py`: Scripts to validate the accuracy of the trained surrogate against the original MCMC.

### Environment & Utilities
* `env/`: A pre-packaged Python virtual environment. It contains all public dependencies required to run the Base SuStaIn simulation immediately (see Quick Start below).

## Requirements & Prerequisites

To run this repository manually from scratch, please install the public dependencies listed in `requirements.txt`. You can install them in a virtual environment using:
```bash
pip install -r requirements.txt
```

### Quick Start (Base Model)
If you want to run the base `run_simulation.py` script without installing anything, you can use the included `env/` folder directly:
```bash
.\env\Scripts\python.exe -u run_simulation.py
```

### Dependency Overview:
* **Base SuStaIn (`pySuStaIn`):** The main library required to run the model is `pySuStaIn`. **This library is already included locally in the `pySuStaIn/` directory** of this repository. You do *not* need to install it from GitHub. As long as you install the dependencies in `requirements.txt` (like `numpy`, `scipy`, `matplotlib`, `sklearn`), the scripts will run smoothly using the local `pySuStaIn` folder.
* **Surrogate Model (Private Library):** The scripts inside `Surrogate_SuStaIn/` require the `bayesian_metamodeling` library. This is a private/internal library and must be installed manually from your internal source. 
> [!IMPORTANT]
> **Environment Merging for the Surrogate:** Because the surrogate scripts need to run *both* the neural network and the original base model (for data extraction and validation), your surrogate environment **must** contain both the `bayesian_metamodeling` library AND all the public packages listed in the base `requirements.txt`. If you are setting up a new environment for the surrogate, be sure to run `pip install -r requirements.txt` inside it!
